# Capability Refactor Decisions

Store durable architectural decisions here when later work would otherwise need to rediscover them.

Decision records should contain:

- Title and status
- Context
- Decision
- Consequences
- Alternatives rejected
- Capabilities or phases affected

Use a numbered name such as `0001-capability-evidence.md`.

Do not record routine implementation steps or temporary work status here. Those belong in the active phase file and `../status.md`.
